启动与界面

Calamity 由主窗口和托盘窗口组成。主窗口用于完整配置；托盘窗口用于快速开关、 查看状态和执行常用操作。

  1. 启动应用后，主窗口会自动加载当前设置并尝试启动 sing-box。
  2. 点击菜单栏图标可以展开托盘窗口。
  3. 关闭主窗口默认只是隐藏，不会退出应用。
  4. 使用托盘中的退出按钮，才会执行完整清理并退出。

托盘操作

状态区

托盘顶部会显示连接状态、当前节点、延迟和实时上下行速度。

常用动作

  • 切换系统代理开关
  • 复制代理地址
  • 切换语言
  • 打开主面板
  • 连接或断开代理
  • 退出应用

当 TUN 模式开启时，系统代理会保持关闭状态；托盘中对应开关会被禁用，避免两套接管方式同时启用。

节点与连接

节点页面用于管理代理节点和分组。当前活跃节点会影响实际连接状态，也会同步显示在托盘和侧边栏中。

  • 节点可以按组组织，支持普通分组和 `urltest` 分组。
  • 切换活跃节点后，应用会重启 sing-box 并同步最新连接状态。
  • 如果当前没有活跃节点，即使 sing-box 正在运行，界面也会显示为未连接。

规则

规则页面用于定义域名、后缀、关键词等匹配条件，以及这些流量的出口策略。未命中的流量会落到 Final 出口。

  • 支持自定义顺序，越靠前的规则越先匹配。
  • 支持为规则选择具体节点、`direct` 或 `reject`。
  • 支持设置 Final 出口，用于处理未命中的流量。
  • 托盘中的站点规则是规则系统的快捷入口，不是另一套独立规则。

DNS

DNS 页面用于管理上游 DNS 服务器、匹配规则和最终服务器。它会直接影响解析路径和 Fake-IP 行为。

  • 可以配置 UDP、DoH、DoT 等常见解析方式。
  • 可以为特定域名、后缀或关键词指定服务器。
  • 当 TUN 模式开启时，运行时会强制使用 Fake-IP。

TUN 模式

TUN 模式用于接管系统层流量。当前实现主要面向 macOS，并且需要管理员权限才能启动。

  1. 在设置页开启 TUN 模式。
  2. 按提示完成管理员授权。
  3. 应用会以带权限的方式启动 sing-box，并接管 TUN 流量。
  4. 退出应用时会尝试完整停止该进程，释放 TUN 接口。

如果你只想用浏览器或指定程序走代理，优先使用系统代理；如果希望更完整地接管系统流量，再启用 TUN。

订阅

订阅页面用于添加、更新和启停订阅源。更新后节点列表会被刷新，相关状态也会同步到前端。

  • 支持手动更新单个订阅或全部订阅。
  • 支持自动更新间隔。
  • 订阅修改后建议确认活跃节点是否仍然存在。

日志与排障

日志页面用于查看 sing-box 和应用侧错误。排障时建议按以下顺序检查：

  1. 确认 sing-box 是否已成功启动。
  2. 确认当前是否有活跃节点。
  3. 确认规则 Final 出口和节点出口是否符合预期。
  4. 确认 DNS 模式是否与当前场景匹配。
  5. 使用 TUN 时，确认管理员授权和退出清理是否正常。
